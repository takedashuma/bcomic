### 3.5 主要 resolver の実装ポイント

#### 3.5.1 comicFolders（新着順一覧）

仕様の根拠となる SQL は下記。日本語キーで作品集約し、MAX(bok_mid) の降順で並べる。

SELECT bok_vch6, bok_vch7,

MAX(bok_txt1), MAX(bok_mid) AS max_bok_mid,

MAX(bok_vch8) AS max_bok_vch8, COUNT(bok_vch8) AS cnt_bok_vch8

FROM tb_bok

WHERE bok_dlday IS NULL

AND bok_vch9 = 'comic'

GROUP BY bok_vch6, bok_vch7

ORDER BY max_bok_mid DESC

LIMIT 24 OFFSET ?;

Prisma での実装は下記。

const groups = await prisma.volume.groupBy({

by: \['authorJa', 'titleJa'\],

where: { deletedAt: null, vch9: 'comic', /\* +曖昧検索 \*/ },

\_max: { id: true, updatedAt: true },

\_count: { \_all: true },

orderBy: { \_max: { id: 'desc' } },

skip, take: pageSize,

});

// 各グループの最新巻と日本語名を別クエリで取得（並列）

items = await Promise.all(groups.map(async (g) =\> {

const latest = await prisma.volume.findFirst({

where: { authorJa: g.authorJa, titleJa: g.titleJa, deletedAt: null, vch9: 'comic' },

orderBy: { id: 'desc' }, // bok_mid 最大 = 最新巻

});

const isFav = await prisma.bookmark.findFirst({

where: { userId, authorJa: g.authorJa, titleJa: g.titleJa, deletedAt: null }

});

return { id: folderId(latest.authorEn, latest.titleEn), ..., isFavorite: \!\!isFav };

}));

#### 3.5.2 toggleFavorite

// URLは英字(vch2,vch3)で受けるが、tb_bkm のキーは日本語(vch6,vch7)に揃える

const ref = await prisma.volume.findFirst({

where: { authorEn, titleEn, deletedAt: null, vch9: 'comic' }

});

// ref.authorJa, ref.titleJa を主キーとして検索

const existing = await prisma.bookmark.findFirst({

where: { userId, authorJa: ref.authorJa, titleJa: ref.titleJa, deletedAt: null }

});

if (existing) {

// 論理削除（既存仕様）

await prisma.bookmark.update({ where: { id: existing.id }, data: { deletedAt: new Date() } });

return false;

} else {

// PK は自動採番でないため MAX+1

const max = await prisma.bookmark.aggregate({ \_max: { id: true } });

await prisma.bookmark.create({ data: { id: (max.\_max.id ?? 0) + 1, ... } });

return true;

}

| Accept-Ranges | bytes（必ず返す）                                                 |

### 3.7 Swagger UI / OpenAPI

  - apps/api/openapi.yaml に OpenAPI 3.1 形式で全エンドポイントを記述。

  - swagger-ui-express で /docs に配信。'Try it out' で実行可能。

  - openapi.yaml の servers リストの先頭は url:/（相対URL）。これによりブラウザで開いた経路（LAN/WAN）に対してリクエストするため、CORS が問題にならない。

  - nginx で /docs /openapi.yaml /openapi.json を api コンテナへプロキシ。

// apps/api/src/index.ts

const openapiSpec = YAML.parse(fs.readFileSync('openapi.yaml','utf8'));

app.use('/docs', swaggerUi.serve, swaggerUi.setup(openapiSpec, {

customSiteTitle: 'zcomic-next API',

swaggerOptions: { persistAuthorization: true },

}));

app.get('/openapi.yaml', (\_q,r) =\> r.type('text/yaml').send(openapiText));

app.get('/openapi.json', (\_q,r) =\> r.json(openapiSpec));
