# zcomic-next 管理側 画面別機能一覧

| 文書ID | ZCN-ADM-FN-001 |
|---|---|
| 対象 | apps/api-admin + apps/web-admin |
| 版数 | 1.0 |
| 発行日 | 2026-05-20 |

旧PHP `192.168.40.99:8002/admin_new/*.php` および `192.168.40.99:8001/V_*` 後継。
本サイト(api/web)とは **完全分離** (別 Cookie / 別 Container / 別 Port)。

| 項目 | 値 |
|---|---|
| API ポート (admin) | 4001 |
| Web ポート (admin) | 8081 |
| Cookie | `zc_admin_token` (HMAC HS256 JWT) |
| 認証元 | `tb_adm` (平文→bcrypt自動移行) |

## 2. ログイン (`/login`)

| 機能 | 振る舞い |
|---|---|
| 名前/パスワード入力 | `tb_adm.adm_vch0` / `adm_vch1` を `mutation adminLogin` に送信 |
| 平文パスワード移行 | 既存 PHP 版は平文保存。初回ログイン時に bcrypt(10) でハッシュ化 |
| 認証成功時 | JWT を `httpOnly` Cookie `zc_admin_token` に Set |
| 最終ログイン日時 | `tb_adm.adm_lgday` を更新 |
| ログアウト | `clearCookie('zc_admin_token')` |

## 3. Home (`/`)

ホーム画面には **重い非同期ジョブ3つ** と **検索結果に対する操作** が並ぶ。

### 3.1 非同期ジョブ (上段)

| ボタン | 動作 | 環境変数 |
|---|---|---|
| **rar / zip 解凍** | `EXTRACT_ARCHIVE_DIR` 配下の `.rar/.zip/.cbz/.7z` を全て解凍 | `EXTRACT_ARCHIVE_DIR` / `UNZIP_DEST_DIR` / `ARCHIVE_DONE_DIR` |
| **話フォルダー結合** | `MERGE_CHAPTER_DIR` 配下の話別フォルダを巻フォルダに結合 | `MERGE_CHAPTER_DIR` |
| **ERComic 解凍** | 上記の成人向け版。別ディレクトリも指定可 | `EXTRACT_ER_ARCHIVE_DIR` 他 |

- 非同期ジョブは `startXxx` で起動 → `jobStatus(id)` で polling
- `JobProgress` コンポーネントでログ/進捗を表示
- 解凍ルール:
  - `title v07.rar` / `title_v07.rar` / `titlev07.rar` → 巻=`07`
  - `title 02s.rar` / `title_02s.rar` / `title02s.rar` → 巻=`02s` (特装版)
  - `title 第03巻.rar` → 巻=`03`
  - `title v01-03.rar` → 範囲扱い → 内側フォルダ名から巻を取得
  - 巻番号は2桁0埋め (≥100 はそのまま)
  - 画像連番は最低3桁0埋め (001..)、>1000 で自動拡張
  - 全角数字→半角 / 全角空白→半角 / 禁則文字(`<>:"/\|?*`)→全角化
  - macOS APFS+virtiofs の NFC/NFD ミスマッチに対応 (`pickAccessiblePath`)

### 3.2 検索結果に対する操作 (下段)

| 列 | 内容 |
|---|---|
| ID | `tb_bok.bok_mid` |
| パス | クリックでパス文字列を検索キーワードに反映して再検索 |
| 巻 | `bok_vch4` (no) / `bok_vch9` (kind) |
| 操作 | 4ボタン (下表参照) |

| 操作ボタン | 動作 | 旧PHP対応 |
|---|---|---|
| **移動** | タイトル配下の全巻を `COMIC_ROOT` → `REGIST_DIR` へ移動 + `tb_bok` の該当全巻レコード削除 | (旧UI 移動) |
| **タイトルフォルダ作成** | `REGIST_DIR/<title>` レベルのフォルダを空で作成 (巻なし) | `admin_new/makeFolder.php` |
| **Delete DB** (赤) | `tb_bok` の該当行のみ削除 (FS無関与) | `admin_new/_DBDelete.php` |
| **Delete DB&Dir** (赤) | `tb_bok` の該当行を削除 + `COMIC_ROOT` のフォルダを rm -rf | `_DBDelete.php` (deleteDir有) |

- 著者/タイトル列は撤廃 (パスから判別可能で冗長のため)
- 移動は `cp -a <src>/. <dst>` シェル経由で **NFC/NFD 耐性** + **コピー完全性検証** (src/dst ファイル数比較 → 不一致なら src 削除せず停止)

## 4. Unknown 検索 (`/unknown`)

旧 `search_unknown.php` + `read_html.php` の合成画面。

### 4.1 Search UnKnown

`REGIST_DIR/0` 配下の Unknown フォルダ一覧。各行は `Title` (フォルダ名全体) と `Link` (日本語タイトルのクリック可能テキスト) のみ表示。

| Linkクリック時の動作 |
|---|
| 日本語タイトルを中段の `日本語タイトル` 入力にセット |
| マンガ王国検索を自動実行 |
| 該当行を黄色ハイライト |

### 4.2 マンガ王国検索

- URL: `https://comic.k-manga.jp/search/word/<keyword>`
- スクレイピング: 旧PHPは UA 未設定。新実装は **2段階リトライ** (1) axios デフォルト UA (2) `curl/7.81.0`
- セレクタ: `a[href*="/title/<digits>"]` 起点に `h2` (title), `p span` (authors), `img` (cover)
- 正規表現フォールバック: cheerio で取れなければ HTML 全文を正規表現で走査
- 失敗時は `(error) ...` / `(no hits)` の診断エントリを返却し、フロントの details で内容確認可能

### 4.3 下段 詳細パネル (選択中の Unknown 行に対して)

| 項目 | 内容 |
|---|---|
| タイトル | マンガ王国 第一候補のタイトル (表示のみ) |
| 検索テキスト | 編集可能 + 「再検索」ボタン |
| 著者 | 編集可能。デフォルト `[;<authors>]` 形式で自動入力 |
| **変更** ボタン | 著者 入力値で対象 Unknown フォルダを rename<br>`/0/[Unknown;Unknown] EnT;JpT` → `/<initial>/<著者> EnT;JpT`<br>initial は EnAuth → JpAuth → `0` の順で先頭1文字から自動算出 |
| LinkPage | マンガ王国 第一候補の `/title/{id}/pv` URL (target=_blank) |
| その他候補 | 折りたたみで第2候補以下を参照可能。各候補に「この著者を採用」 ボタン |

## 5. 比較標準 (`/compare-normal`) / 比較E (`/compare-ero`)

旧 `comparNo.php?mode=COMIC` (or `=ER`) 後継。

### 5.1 上段操作

| ボタン | 動作 |
|---|---|
| **比較取得** | `REGIST_DIR` を深さ3でスキャン → 葉フォルダ一覧 + 同タイトルの DB 既存巻 (stock) |
| **NormalComic登録** | `startRegistUnregistAll` 非同期ジョブを起動 (REGIST_DIR の全エントリを tb_bok に登録 + COMIC_ROOT にコピー + REGIST_DIR を空にする) |

### 5.2 結果テーブル

| 列 | 内容 |
|---|---|
| # | 連番 |
| REGIST_DIR パス | 葉フォルダパス `/initial/[Author;著者] Title;タイトル/volume` |
| 既存巻 (stock) | 同タイトルでDBに既存の全巻リスト。1巻毎に下記ボタン |
| 行操作 | 変更ボタン (フォルダ rename) |

### 5.3 stock 1巻ごとのボタン

| ボタン | 動作 | 旧PHP対応 |
|---|---|---|
| **入換** | `REGIST_DIR/<stock.path>` → `COMIC_ROOT/<stock.path>` に入換 (rm dst + copy + rm src) | `admin_new/exchangeDir.php` |
| **削除** (赤) | `tb_bok` 行 + `COMIC_ROOT/<stock.path>` を削除 | `admin_new/deleteDBandBook.php` |

完了済みの stock 行は灰色 + 取消線でフィードバック。

## 6. Page 取得 (`/search-page`)

13dl.me のカテゴリページから新着タイトル + RapidGator リンクをクロール。

| 項目 | 内容 |
|---|---|
| カテゴリURL | デフォルト `https://13dl.me/category/raw-manga/` (編集可) |
| ページ番号 | 数値入力 (1〜) |
| 範囲 | 3チャンク選択: **Page:1-7 / Page:8-15 / Page:16-24** (1ページ24タイトルを3分割) |
| クロール出力 | 各アイテム: タイトル / 詳細URL / stock (DB既存巻CSV) / foundNo (検出巻番号) / newDir or existDir / RapidGater リンク一覧 |
| **ExistDir** ボタン | 既存 COMIC フォルダの相対パスを `REGIST_DIR` に空で作成 (mkdir -p) |
| **NEW Dir** ボタン | 新規登録用フォルダ `/0/[Unknown;Unknown] EN;JP` を `REGIST_DIR` に空で作成 |
| RapidGator | 抽出された全 RG リンクを表示。クリックで別タブ |

## 7. 共通仕様

### 7.1 認証 (httpOnly Cookie + JWT)

- Cookie 名: `zc_admin_token`
- 本サイトの `zc_token` とは **完全分離** (Cookie 名/JWT_SECRET/コンテナ独立)
- 有効期限: `ADMIN_JWT_EXPIRES_IN` (デフォルト1d)
- 検証は Apollo Server context で毎リクエスト

### 7.2 非同期ジョブ

すべて **in-memory job store** + **polling**:

```
mutation startXxx → { id, status: 'running' }
↓ polling 1-3秒間隔
query jobStatus(id) → { status: 'success'|'failed', logs, outputs, elapsedSec }
```

ジョブログは UI の `<JobProgress>` でリアルタイム表示 (累積 logs)。

### 7.3 ファイルシステム操作

| 操作 | 実装 |
|---|---|
| 再帰スキャン | `fs.readdir` (`@eaDir`, `.DS_Store`, `^DS_|@` を除外) |
| 移動 (同一FS) | `fs.rename` (アトミック) |
| 移動 (cross-device) | `cp -a <src>/. <dst>` シェル + `rm -rf src` + ファイル数検証 |
| 削除 | `fs.rm({ recursive:true, force:true })` |
| 退避 (削除前) | デフォルトで `.__trash/<name>__<timestamp>` に rename |

### 7.4 NFC/NFD 対応

macOS APFS + Docker virtiofs の Unicode 正規化問題に対応:

- `pickAccessiblePath()` — `p` / `p.NFC()` / `p.NFD()` の3バリアントで `fs.access` を試行
- `readdirSafe()` — 同様に readdir を3バリアント
- `statSafe()` — 同様に stat を3バリアント
- Cross-device コピーは coreutils の `cp -a` に任せる (Node JS の `fs.copyFile` より堅牢)

### 7.5 セキュリティ

- すべてのパス操作は `resolveUnderRoot(root, rel)` で root 外を厳格に拒否
- mutation は `requireAdmin(ctx)` でログイン強制
- CORS は `ADMIN_CORS_ORIGINS` で許可 origin を制限

## 8. 主な GraphQL Mutation/Query 一覧 (要約)

| Operation | 用途 |
|---|---|
| `mutation adminLogin(name, password)` | ログイン |
| `mutation adminLogout` | ログアウト |
| `mutation adminChangePassword(old, new)` | パスワード変更 |
| `query me` | ログイン状態 / 管理者情報 |
| `query searchComics(q, page, pageSize)` | tb_bok 部分一致検索 |
| `mutation startExtractAllArchives` | rar/zip 解凍ジョブ起動 |
| `mutation startExtractAllErArchives` | 成人向け 解凍ジョブ起動 |
| `mutation startMergeAllChapters` | 話フォルダー結合ジョブ起動 |
| `query jobStatus(id)` | ジョブ進捗 polling |
| `query listUnknownFolders` | REGIST_DIR/0 一覧 |
| `mutation searchFromMangaKingdom(titleJa)` | マンガ王国検索 |
| `mutation renameRegistFolder(oldDir, newDir, inRegist)` | フォルダ rename |
| `query compareUnregist` | REGIST_DIR 葉一覧 + stock |
| `mutation exchangeDir(newDir)` | REGIST_DIR → COMIC_ROOT 入換 |
| `mutation deleteDBandBook(bookPath)` | tb_bok 行 + COMIC_ROOT 削除 |
| `mutation startRegistUnregistAll` | UNREGIST 一括登録ジョブ起動 |
| `mutation crawl13dlList(...)` | 13dl カテゴリクロール |
| `mutation makeRegistDir(dir)` | REGIST_DIR に空フォルダ作成 |
| `mutation moveToRegist(folderPath)` | COMIC_ROOT → REGIST_DIR 移動 + tb_bok 該当全巻削除 |
| `mutation createRegistFolder(folderPath)` | REGIST_DIR に同名フォルダ作成 |
| `mutation deleteVolumeDB(id)` | tb_bok 該当行のみ削除 |
| `mutation deleteVolumeDBAndDir(id, folderPath)` | DB行 + FS フォルダ両削除 |
