# zcomic-next ビューワー側 画面別機能一覧

| 文書ID | ZCN-FN-001 |
|---|---|
| 対象 | apps/api + apps/web (一般ユーザー向け) |
| 版数 | 1.0 |
| 発行日 | 2026-05-20 |

旧PHP `192.168.40.99:8003/list/index.php?mode=COMIC` 後継のWebアプリ。
画面数は6つ。**JWT(httpOnly Cookie)認証** + **GraphQL** + **画像 HTTP streaming**で構成。

---

## 1. 画面一覧

| # | 画面 | パス | 認証要否 | 概要 |
|---|---|---|---|---|
| 1 | ログイン | `/login` | 不要 | メール/パスワード認証。平文→bcrypt 自動移行 |
| 2 | 新着順一覧 | `/` | 要 | 日本語タイトル単位 GROUP BY の新着順 |
| 3 | お気入一覧 | `/favorites` | 要 | tb_bkm に登録した作品の最新巻一覧 |
| 4 | 設定 | `/settings` | 要 | パスワード変更 / ログアウト |
| 5 | 詳細 | `/title/:id` | 要 | 1作品の全巻リスト + 1ページ目サムネ |
| 6 | ビューワー | `/viewer/:volumeId` | 要 | スライド形式の本文ページ表示 |

## 2. 画面別の詳細機能

### 2.1 ログイン (`/login`)

| 機能 | 振る舞い |
|---|---|
| メール/パスワード入力 | `usr_vch0` (email) と `usr_vch1` (password) を `mutation login` に送信 |
| 平文パスワード移行 | 既存 PHP 版は平文保存。初回ログイン時に bcrypt(10) でハッシュ化して `tb_usr.usr_vch1` を UPDATE |
| 認証成功時 | JWT (uid) を生成し `httpOnly` Cookie `zc_token` に Set (有効期限7日) |
| `usr_lgday` 更新 | ログイン時刻を `tb_usr.usr_lgday` に保存 |
| 認証失敗 | `INVALID_CREDENTIALS` を返却。フォームにエラー表示 |
| ログイン後遷移 | 直前パスがあれば復帰、なければ `/` |

### 2.2 新着順一覧 (`/`)

| 機能 | 振る舞い |
|---|---|
| グリッド表示 | 1作品 = 1カード。3〜6カラムのレスポンシブ |
| 並び順 | `GROUP BY authorJa,titleJa ORDER BY MAX(bok_mid) DESC WHERE vch9='comic' AND deletedAt IS NULL` |
| ページネーション | 24件/ページ。下部に数値ページネーター |
| カード表示内容 | 1ページ目サムネ / 日本語タイトル / 著者 / 最新巻番号 |
| サムネ | `/img/page/:volumeId/1?w=300` (Range / sharp resize) |
| **お気入トグル** | カード本文のハートボタンで `tb_bkm` 追加/削除 (連打耐性: pending state + idempotent transaction) |
| 検索ボックス | 著者/タイトル部分一致。**IME対応** (composition events) + Enter 即時実行 |
| カードクリック | 詳細 `/title/:id` に遷移 |
| URL状態保持 | `?q=…&page=…` で再ロード後も状態復元 |
| ボトムナビ | 「新着順」「お気入」「設定」3アイコン。アクティブハイライト |

### 2.3 お気入一覧 (`/favorites`)

| 機能 | 振る舞い |
|---|---|
| 表示元 | `tb_bkm` (お気入)を起点に `tb_bok` をJOIN |
| **504対策** | 旧版は `tb_bok` から OR で大量条件 → tb_bkm 起点に逆転して高速化 |
| 並び順 | 追加日順降順 |
| 表示内容 | 新着順と同じカード形式 (タイトル/著者/最新巻/サムネ) |
| お気入解除 | カードのハートボタンで即削除 (`prisma.$transaction` で idempotent UPDATE) |
| 0件時 | "お気に入りはまだありません" 案内 |

### 2.4 設定 (`/settings`)

| 機能 | 振る舞い |
|---|---|
| 現ユーザー表示 | `me` query で `tb_usr` の nickname / email を表示 |
| パスワード変更 | 旧パスを確認 → 新パスを bcrypt(10) で保存。`usr_upday` 更新 |
| ログアウト | `clearCookie('zc_token')` + `/login` リダイレクト |

### 2.5 詳細 (`/title/:id`)

| 機能 | 振る舞い |
|---|---|
| 表示元 | `tb_bok` の同 `authorJa+titleJa` グループの全巻 |
| 並び順 | `no ASC` (巻番号昇順) |
| 巻毎の表示 | 巻番号 / 1ページ目サムネ / 巻フォルダ名 |
| 巻クリック | ビューワー `/viewer/:volumeId` へ遷移 |
| お気入トグル | 詳細上部にハートボタン |

### 2.6 ビューワー (`/viewer/:volumeId`)

| 機能 | 振る舞い |
|---|---|
| スライド形式 | 左右クリック / キーボード ← → でページ送り |
| 全ページ取得 | `volumeDetail` query で `fs.readdir` した画像ファイル名リストを返却 |
| 画像配信 | `/img/page/:volumeId/:idx` (Range request対応 / sharp resize 任意) |
| 連続スクロール | スクロールに応じて Range で部分取得 |
| 戻る | 詳細 `/title/:id` へ |

## 3. 共通仕様

### 3.1 認証 (httpOnly Cookie + JWT)

- Cookie 名: `zc_token` (管理サイトの `zc_admin_token` と完全分離)
- 有効期限: `JWT_EXPIRES_IN` (デフォルト7日)
- 検証: Apollo Server / Express の `context` フックで毎回 JWT.verify
- 不正/期限切れ: `null` を返し `me` が `null` で再ログイン誘導

### 3.2 ファイル配信

- パス: `/img/page/:volumeId/:idx?w=N`
- ストリーミング: `fs.createReadStream({start,end})` + `Content-Range`
- リサイズ: `?w=300` 等で `sharp` リサイズ (サムネ)
- セキュリティ: パスは必ず `resolveUnderRoot(COMIC_ROOT, folderPath)` で root外を弾く

### 3.3 URL状態保持

- 一覧系画面 (`/`, `/favorites`) は `?q=…&page=…` を真の状態源 (single source of truth)
- ボトムナビ復帰時は `sessionStorage` に JSON 保存して型を保持
  (旧バグ: `String(v)` で number → string 化により `?page="3"` が出力されていた)

### 3.4 国際化 / 文字化け対策

- DB は utf8mb4 / collation: `utf8mb4_unicode_ci`
- ファイル名 NFC/NFD ミスマッチ対策は管理側 (extract job) に集約。ビューワー側は読み取りのみ
- IME (日本語入力) 検索ボックス: `onCompositionStart/End` + 300ms debounce + Enter 時 `e.nativeEvent.isComposing` チェック

---

## 4. 非機能

| 項目 | 内容 |
|---|---|
| ブラウザ要件 | 直近2世代の Chrome / Safari / Firefox / Edge |
| レスポンス時間 | 一覧 < 1秒 / サムネ初回 < 2秒 / ビューワーページ送り < 200ms |
| 同時接続 | 想定数十名 (社内利用) |
| ログ保持 | コンテナ stdout (Docker logging driver) |
