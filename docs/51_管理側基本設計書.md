## 1. システム概要

zcomic-next 管理サイトは、コミック管理 NAS 上の作業を Web UI で行うための管理ツールである。旧PHP 版 (`/admin_new/*.php`) と旧 Vite 版 (`V_*`) を Node.js + GraphQL + React に置き換え、Docker Compose で本サイト (一般ユーザー向け) と完全分離して運用する。

管理者は社内 LAN から `http://192.168.40.99:8081/` にアクセスし、tb_adm に登録されたアカウントでログインする。Cookie 名・JWT 秘密鍵・コンテナ・ポートはすべて本サイトと分離。

---

## 2. 機能要件

### 2.1 認証

- tb_adm の name / password でログイン (パスワードは平文→bcrypt 自動移行)
- ログイン状態は httpOnly Cookie `zc_admin_token` に JWT を格納
- 最終ログイン日時を `tb_adm.adm_lgday` に記録
- 管理者パスワード変更画面 (旧パス確認 → 新パス bcrypt 保存)

### 2.2 6画面の主機能

| 画面 | 主機能 |
|---|---|
| Home | rar/zip 解凍 / 話フォルダー結合 / ERComic 解凍 / 検索結果に対する移動・タイトル作成・DB削除・DB&Dir削除 |
| Unknown検索 | REGIST_DIR/0 の Unknown フォルダ一覧 / マンガ王国検索 / 著者入力で rename |
| 比較標準 | REGIST_DIR 葉一覧と tb_bok stock 突合 / stock 1巻毎の入換・削除 / NormalComic 一括登録 |
| 比較E | 比較標準の成人向け版 |
| Page取得 | 13dl.me カテゴリページのクロール / ExistDir / NEW Dir 作成 |
| ログイン | tb_adm 認証 / パスワード変更 |

### 2.3 解凍ジョブの命名ルール

- `Title v07` / `Title_v07` / `Titlev07` → 巻=`07` (区切り任意)
- `Title 02s` / `Title_02s` / `Title02s` → 巻=`02s` (特装版命名)
- `Title 第03巻` → 巻=`03`
- `Title v01-03` などの範囲表記 → 巻は内側フォルダから取得
- 巻番号は2桁0埋め (≥100 はそのまま)、画像連番は最低3桁0埋め
- 全角数字→半角、全角空白→半角、禁則文字 `<>:"/\|?*` → 全角化

### 2.4 ファイルシステム操作

- 移動 (同一FS): `fs.rename` でアトミック
- 移動 (cross-device): `cp -a <src>/. <dst>` + ファイル数検証 + `rm -rf src`
- NFC/NFD ミスマッチ対策: `pickAccessiblePath` / `readdirSafe` / `statSafe` で3バリアント試行
- 削除はデフォルト `.__trash` 退避、`permanent=true` で物理削除

## 3. 非機能要件

| 項目 | 要件 |
|---|---|
| 可用性 | 社内LAN 24時間/平日 |
| 性能 | 検索 < 1秒 / 移動・削除 < 2秒 / 解凍は数秒〜数十分 (非同期) |
| 同時利用 | 管理者 1〜数名想定 |
| セキュリティ | JWT (httpOnly) / `requireAdmin` / `resolveUnderRoot` でパス制限 |
| ログ | Docker stdout |
| バックアップ | DB は既存 NAS の DB バックアップに準拠 |

---

## 4. 制約条件

- DB は既存 MariaDB の comic スキーマ (tb_bok, tb_bkm, tb_usr, tb_adm) をそのまま利用
- ファイル配置は既存命名規則 `/<initial>/[Author;著者] Title;タイトル/<volume>/` を踏襲
- Mac APFS + Docker virtiofs の NFC/NFD 不整合に対応
- 外部スクレイピングは k-manga.jp と 13dl.me。サイト構造変化への追従が必要

---

## 5. 外部インタフェース

| 相手先 | プロトコル | 用途 |
|---|---|---|
| MariaDB | MySQL (TCP 3306) | Prisma 経由 |
| NAS ファイルシステム | Docker bind mount | COMIC_ROOT / STAGING_ROOT / DOWNLOAD_ROOT |
| comic.k-manga.jp | HTTPS | マンガ王国検索 (UA: axios → curl の2段階) |
| 13dl.me | HTTPS | カテゴリページクロール |

---

## 6. ユーザー層

社内の専任管理者数名のみ。Webスキル中〜高。コミック整理業務に習熟しており、旧PHP UI からの移行が前提。
