## 3. 詳細

### 3.1 ページ状態URL保持

- `?q=` / `?page=` を TanStack Router のsearch params として扱う
- ブラウザ戻る/進む / 別タブ再現 / 共有可能なリンクを実現
- ボトムナビからの復帰時は sessionStorage に JSON で保存し `JSON.parse` で型を維持

### 3.2 お気入の主体化 (tb_bkm)

- 旧版は `tb_usr.usr_vchN` を CSV で持っていたが、tb_bkm を正式な主データに昇格
- お気入トグルは `optimisticResponse` + `cache.modify` で即時反映 + サーバ `updateMany` で idempotent UPDATE
- Favorites 一覧は tb_bkm 起点 (LIMIT/OFFSET) → tb_bok を IN で JOIN するクエリに最適化
  (従来は tb_bok から大量 OR 検索 → 504 が頻発)

### 3.3 IME対応

- 検索ボックスを controlled component にしつつ、`isComposing` 中は親へ反映しない
- `compositionend` 後に 300ms debounce してから検索キーワードを更新
- Enter キーは `e.nativeEvent.isComposing` が true の場合スキップ (確定時の意図しない検索を防ぐ)

### 3.4 ボトムナビ

- `useLocation` でリアクティブに pathname を取得し、アクティブ判定
- sessionStorage の保存値は `JSON.stringify(state)` でオブジェクトをそのまま保存
  (誤って `String(v)` で number→string 化していたバグを修正)

### 3.5 外部DDNS / 相対URL運用

- `VITE_API_URL=""` (空) を許容するため docker-compose で `${VITE_API_URL-}` (コロンなし) を使う
- apollo client は相対URLで `/graphql` を叩く
- LAN: `http://192.168.40.99:8080/` → nginx → api
- 外部: `https://dirs.synology.me` → Synology Reverse Proxy → 同上

---

## 4. 非機能の追加要件

- nginx `proxy_read_timeout` を 600s に延長 (管理側 / 大きい画像配信)
- Sharp によるサムネ生成は CPU バウンドのため、容量制限と並列制限を設けてホストを保護
- FS 読み取り (画像) はサンドボックス: `resolveUnderRoot(COMIC_ROOT, …)` で root 外を厳格に弾く
