## 1. 全体構成

管理サイトは Node.js (Apollo Server v4 + Express + Prisma) を **api-admin** として、Vite + React + Apollo Client を **web-admin** として独立コンテナで動作する。本サイトとは Cookie/Secret/Port を分離し、ユーザー側と相互影響しない。

## 3. 主要処理フロー

### 3.1 一括解凍 (`startExtractAllArchives`)

- `EXTRACT_ARCHIVE_DIR` を再帰 readdir で `.rar/.zip/.cbz/.7z` 列挙
- `parseArchiveBaseName(archiveBase)` で `{title, volume}` を抽出
- 一時ディレクトリに `7z` または `unrar-free` で解凍
- `detectLayout` でラッパー1段以上深い構造でも対応
- `layout=single` なら 1巻直 / `multi` なら inner フォルダごとに `parseInnerFolderToVolume`
- `padImage` で連番 0埋め (最低3桁、>1000 で自動拡張)
- `moveFileResilient` で rename / copyFile+unlink にフォールバック
- 元アーカイブを `ARCHIVE_DONE_DIR` に移動 (同名は `__<timestamp>` 付加)

### 3.2 cross-device 安全移動 (`moveToRegist`)

- `fs.rename` を試行 (同一FSなら高速)
- EXDEV ⇒ `runCmd('cp', ['-a', src + '/.', dst])` で堅牢コピー
- `countFilesRecursive` で src/dst のファイル数を比較
- dst < src なら src を残してエラー停止 (データ消失防止)
- 完全に一致したら `rm -rf src`
- tb_bok を `folderPath startsWith <title>/` で `deleteMany`

### 3.3 stock 入換 (`exchangeDir`)

- `COMIC_ROOT/<newDir>` を `rm -rf`
- `REGIST_DIR/<newDir>` → `COMIC_ROOT/<newDir>` へ `copyRecursive`
- `REGIST_DIR/<newDir>` を `rm -rf`

### 3.4 Unknown 変更 (`renameRegistFolder`)

- 対象は REGIST_DIR 内 (`inRegist=true`) または COMIC_ROOT (`inRegist=false`)
- 旧 oldDir → 新 newDir に `fs.rename`
- 親ディレクトリは事前に `mkdir -p` (initial が変わるケース)

### 3.5 マンガ王国検索

- URL: `https://comic.k-manga.jp/search/word/<encoded keyword>`
- UA: 1) axios デフォルト (旧 PHP curl 相当) 2) `curl/7.81.0`
- レスポンスが 5KB未満 かつ `/title/<id>` 未含有なら 2回目を試行 (SPA stub対策)
- セレクタ: `a[href*="/title/<digits>"]` 起点に `h2` / `p span` / `img`
- cheerio で取れない時は HTML 全文を正規表現で `<a href="/title/<id>/pv">…</a>` 走査
- 失敗時は `(error) ...` / `(no hits)` 診断エントリを返却。フロントの details で内容確認可能

## 4. DB アクセス (Prisma)

- `Volume` (tb_bok) は `@map` で legacy カラム (bok_mid, bok_vch0..9, bok_txt1) を保持
- ID 採番は手動 MAX+1 (既存PHP仕様)
- 削除は基本 `deletedAt` セットで soft delete。物理 delete は限定 (`deleteVolumeDB` / `deleteDBandBook`)
- 検索は `contains` で部分一致。tb_bok から OR では遅いため tb_bkm 起点逆引きを併用

---

## 5. UI / フロント設計

### 5.1 ルーティング (TanStack Router)

- `/login` / `/` / `/unknown` / `/compare-normal` / `/compare-ero` / `/search-page`
- 未認証時は `/login` にリダイレクト

### 5.2 Apollo Client

- baseURL は相対 URL (nginx 同一オリジン)
- cache-policy = `network-only` を多用 (操作の即時反映)
- `refetchQueries` で連動更新

---

## 6. エラーハンドリング

- Mutation は `{ ok, message, path }` を返却。フロントは `ok==false` で赤バー表示
- ジョブは `status==failed` + logs で UI 表示
- API 例外は GraphQL errors で返却 (UNAUTHENTICATED / INVALID_CREDENTIALS など)
