## 3. リクエストフロー

### 3.1 GraphQL クエリ（同一オリジン経由）

```mermaid
sequenceDiagram
    autonumber
    actor User as ユーザー
    participant Browser as ブラウザ
    participant Nginx as web/nginx
    participant API as api/ApolloServer
    participant Prisma as Prisma
    participant DB as MariaDB

    User->>Browser: 一覧画面アクセス
    Browser->>Nginx: GET / (HTML)
    Nginx-->>Browser: index.html + JS bundle
    Browser->>Nginx: POST /graphql<br/>{ query: comicFolders }<br/>Cookie: zc_token
    Nginx->>API: proxy_pass http://api:4000
    API->>API: readAuthCookie → JWT verify
    API->>Prisma: groupBy(authorJa,titleJa)
    Prisma->>DB: SELECT ... GROUP BY ...
    DB-->>Prisma: 24 rows
    Prisma-->>API: items
    API->>Prisma: 各グループの最新巻取得 (Promise.all)
    Prisma->>DB: SELECT ... ORDER BY id DESC
    DB-->>Prisma: latest volumes
    API-->>Nginx: JSON response
    Nginx-->>Browser: JSON
    Browser->>Browser: render grid
```

### 3.2 画像配信 (Range Request)

```mermaid
sequenceDiagram
    autonumber
    participant Browser as ブラウザ
    participant Nginx as web/nginx
    participant API as api/Express
    participant FS as ファイルシステム

    Browser->>Nginx: GET /img/page/123/5<br/>Range: bytes=0-65535<br/>Cookie: zc_token
    Nginx->>API: proxy_pass
    API->>API: JWT verify
    API->>API: tb_bok から folderPath 取得
    API->>API: resolveVolumeFolder (パス安全性)
    API->>FS: fs.readdir(folder)
    FS-->>API: ['001.jpg', '002.jpg', ...]
    API->>FS: fs.statSync('005.jpg')
    FS-->>API: size=523456
    API->>FS: createReadStream({start:0, end:65535})
    API-->>Nginx: 206 Partial Content<br/>Content-Range: bytes 0-65535/523456
    Nginx-->>Browser: 206 (stream)
```

### 3.3 ログイン + 平文パスワード移行

```mermaid
sequenceDiagram
    autonumber
    participant Browser as ブラウザ
    participant API as api/Apollo
    participant DB as MariaDB

    Browser->>API: mutation login(email, password)
    API->>DB: SELECT * FROM tb_usr WHERE usr_vch0=email AND usr_dlday IS NULL
    DB-->>API: user (usr_vch1='plain_password')
    API->>API: stored.startsWith('$2') ? NO
    API->>API: plain比較 OK
    API->>API: bcrypt.hash(given, 10)
    API->>DB: UPDATE tb_usr SET usr_vch1='$2a$10$...' WHERE usr_mid=...
    API->>DB: UPDATE tb_usr SET usr_lgday=NOW()
    API->>API: jwt.sign({ uid })
    API-->>Browser: Set-Cookie: zc_token=...<br/>{ id, email, nickname }
```

## 5. ネットワーク要件

| 接続元 | 接続先 | プロトコル/ポート | 用途 |
|---|---|---|---|
| クライアントPC | 192.168.40.99:8080 | HTTP | 内部アクセス |
| クライアントPC | dirs.synology.me:443 | HTTPS | 外部アクセス（Synology Reverse Proxy） |
| Synology DSM | 192.168.40.99:8080 | HTTP | 内部転送 |
| api コンテナ | 192.168.40.99:3306 | MySQL/TCP | DB接続 |
| api コンテナ | /Volumes/public/Comic/Renamed | ファイルIO | bind mount |
| api コンテナ | (外部不要) | — | — |
