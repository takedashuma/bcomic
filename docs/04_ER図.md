## 2. 物理ER図

```mermaid
erDiagram
    tb_usr ||--o{ tb_bkm : "usr_mid = bkm_uid"
    tb_usr ||--o{ tb_red : "usr_mid = red_uid"
    tb_bok ||--o{ tb_red : "bok_mid = red_bid"
    tb_bok }o--o{ tb_bkm : "vch6,vch7 で一致 (DB制約なし)"

    tb_usr {
        int usr_mid PK
        varchar usr_vch0 "email"
        varchar usr_vch1 "password"
        varchar usr_vch2 "nickname"
        datetime usr_lgday
        datetime usr_dlday
    }
    tb_bok {
        int bok_mid PK
        varchar bok_vch0 "TopFolder"
        varchar bok_vch2 "Author英"
        varchar bok_vch3 "Title英"
        varchar bok_vch4 "No"
        varchar bok_vch6 "著者"
        varchar bok_vch7 "タイトル"
        varchar bok_vch8 "巻数"
        varchar bok_vch9 "種別"
        text bok_txt1 "FolderPath"
        datetime bok_upday
        datetime bok_dlday
    }
    tb_bkm {
        int bkm_mid PK
        int bkm_uid FK
        varchar bkm_vch2 "Author英"
        varchar bkm_vch3 "Title英"
        varchar bkm_vch6 "著者"
        varchar bkm_vch7 "タイトル"
        datetime bkm_dlday
    }
    tb_red {
        int red_mid PK
        int red_uid FK
        int red_bid FK
        int red_int0 "LastPage"
        datetime red_upday
    }
```

## 3. リレーション解説

### 3.1 USERS — BOOKMARKS（1:N）

`tb_bkm.bkm_uid` は `tb_usr.usr_mid` を参照（FK定義は無いが論理FK）。1ユーザーが多数のお気入を持つ。

### 3.2 USERS — READING_PROGRESS（1:N）

`tb_red.red_uid` は `tb_usr.usr_mid` を参照（明示FK + ON DELETE CASCADE）。

### 3.3 VOLUMES — READING_PROGRESS（1:N）

`tb_red.red_bid` は `tb_bok.bok_mid` を参照（明示FK + ON DELETE CASCADE）。

### 3.4 VOLUMES — BOOKMARKS（M:N 論理）

直接のFK関係は無く、`(authorJa, titleJa)` の組で論理的に紐づく。

- お気入は **作品単位**（複数巻を含む）
- 同じ作品の複数巻は同じ `(authorJa, titleJa)` を共有

## 4. 重要な制約

| 制約 | 内容 |
|---|---|
| 論理削除 | `*_dlday IS NULL` のみ有効レコードとして扱う |
| 作品の同一性 | `(authorJa, titleJa) = (bok_vch6, bok_vch7)` で同定 |
| 種別フィルタ | `bok_vch9 = 'comic'` のみ本システムで扱う |
| tb_bkm 自動採番なし | INSERT 時に `MAX(bkm_mid) + 1` で採番 |
| tb_red 自動採番あり | `red_mid INT AUTO_INCREMENT` |
| tb_red ユニーク | `UNIQUE (red_uid, red_bid)` でユーザー×巻一意 |
