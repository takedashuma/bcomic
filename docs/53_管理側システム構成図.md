## 3. ボリュームマウント

| 環境変数 (ホスト) | コンテナパス | モード | 用途 |
|---|---|---|---|
| `COMIC_ROOT_HOST` (/Volumes/public/Comic/Renamed) | `/comics` | **rw** | 入換・削除のため書き込み必要 |
| `STAGING_ROOT_HOST` (= COMIC_ROOT_HOST) | `/staging` | rw | フォルダ移動・作成の作業領域 |
| `DOWNLOAD_ROOT_HOST` (/Volumes/public/Download) | `/download_root` | rw | `/Zip /Unzip /zFin /Chapter /Regist` を内包 |

DOWNLOAD_ROOT をまとめて1マウントしているのは、`/Zip → /Unzip → /Regist → /comics` のフォルダ移動が **cross-device EXDEV** にならないようにするため。

## 4. リクエストフロー (代表3パターン)

### 4.1 一括解凍ジョブの起動 → ポーリング

```mermaid
sequenceDiagram
    autonumber
    actor Admin
    participant Web as web-admin
    participant API as api-admin
    participant FS
    participant DB

    Admin->>Web: 「rar/zip 解凍」クリック
    Web->>API: mutation startExtractAllArchives
    API->>API: runJobAsync (in-memory store)
    API-->>Web: { id: "job-xxx", status: running }
    loop 1〜3秒間隔
        Web->>API: query jobStatus(id)
        API-->>Web: { status, logs, elapsedSec }
    end
    par 解凍ワーカー
        API->>FS: spawn 7z / unrar-free
        FS-->>API: 解凍ファイル
        API->>FS: 命名規則でリネーム
        API->>FS: rar/zip を ARCHIVE_DONE_DIR に退避
    end
    API->>API: status=success
    Web->>API: 最後の jobStatus
    API-->>Web: { status: success, outputs: [...] }
```

### 4.2 比較取得 → stock 入換

```mermaid
sequenceDiagram
    autonumber
    participant Admin
    participant Web
    participant API
    participant FS
    participant DB

    Admin->>Web: 「比較取得」
    Web->>API: query compareUnregist
    API->>FS: REGIST_DIR を深さ3スキャン
    API->>DB: 各エントリ titleJa contains で同タイトル巻取得
    API-->>Web: entries[].stockBooks[]

    Admin->>Web: stock 1巻 ⇒ 「入換」
    Web->>API: mutation exchangeDir(newDir = stock.folderPath)
    API->>FS: rm COMIC_ROOT/<path>
    API->>FS: cp -a REGIST_DIR/<path> → COMIC_ROOT/<path>
    API->>FS: rm REGIST_DIR/<path>
    API-->>Web: { ok, message }
```

### 4.3 検索結果 → タイトル全巻移動 + DB全巻削除

```mermaid
sequenceDiagram
    autonumber
    participant Admin
    participant Web
    participant API
    participant FS
    participant DB

    Admin->>Web: Search → 「移動」
    Web->>Web: titleFolderPath で /<volume> を除去
    Web->>API: mutation moveToRegist(folderPath)
    API->>FS: fs.rename src → dst
    alt EXDEV (cross-device)
        API->>FS: cp -a src/. dst
        API->>FS: countFilesRecursive src/dst を比較
        alt 件数一致
            API->>FS: rm -rf src
        else 件数不一致
            API-->>Web: ok=false (src 保持で停止)
        end
    end
    API->>DB: deleteMany where folderPath startsWith <title>/
    API-->>Web: { ok, message: "移動 (N/N files) / tb_bok M件削除" }
```

## 6. 認証経路

```mermaid
flowchart LR
    U[未ログイン] -->|/login へ| L[ログイン画面]
    L -->|adminLogin OK| C[Cookie zc_admin_token]
    C -->|/ へ| Auth[認証済レイアウト]
    Auth --> Me[me query]
    Me -->|me=null| U
    Me -->|me={...}| Home[Home / Unknown / Compare / Page]
    Home -->|logout| Cl[clearCookie]
    Cl --> U
```

> Cookie `zc_admin_token` は本サイトの `zc_token` と完全別物。
> ブラウザ的にも path/scope が異なり、本サイトの認証状態とは完全に独立。
