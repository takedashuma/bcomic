### マンガ王国検索パーサの強化

旧PHPは `body > div > div > div > section > section > div > ul > li` の positional 指定で
取得していたが、k-manga.jp の DOM 構造変化に弱い。本実装では:

- 起点を `a[href*="/title/"]` (より厳密には `/title/{digits}` を含む `<a>`) にして堅牢化
- 各 `<a>` から `h2` でタイトル、`p span` で著者を抽出。`alt` 属性もフォールバック
- 同URLは重複排除
- 旧PHPと同じ positional セレクタ (`body > div > div > div > section section > div > ul > li`) もフォールバックとして併用
- 取得失敗時は `(error) ...` / `(no hits)` の診断エントリを返却し、
  フロント側で内容を `details` で確認できるよう表示
- HTTP ステータス 4xx/5xx も診断エントリとして返却 (validateStatus 無効化)
- User-Agent / Accept-Language を一般的なブラウザ寄りに設定

### Chrome UA で SPA stub が返る問題への対応

旧PHP curl は UA を明示設定せず (curl デフォルト) でアクセスしていた。
新実装で Chrome 系 UA を付けたところ、k-manga 側で JS-render 想定の SPA stub HTML
(約 2.5KB / `/title/<id>` 链接無し) が返るケースを確認。

対応:

- **2段階リトライ**: 1) axios デフォルト UA → 2) `curl/7.81.0`。最初のレスポンスに
  `/title/<id>` が含まれず HTML が 5KB 未満なら次の UA で再試行。
- **正規表現フォールバック**: cheerio で候補が取れなかった場合、HTML 文字列を直接
  `<a href="/title/<id>/pv">...</a>` ブロックの正規表現で走査し、内部の `h2`/`span`/`img` から
  タイトル・著者・カバーを抽出。
- **診断情報の充実**: (no hits) 時の description に HTML 先頭 800 文字を含める
  (空白圧縮)。何が返ってきたかフロントの details で確認可能。

## 移動: タイトル配下を一括 & 中身ごと確実にコピー (cp -a)

### 不具合
旧実装 (`copyRecursive` を JS の `fs.readdir/copyFile` で組んだもの) では、
日本語 NFC/NFD ミスマッチが起きると **巻フォルダだけ作成され、中の画像が
コピーされない** 事象が発生していた (Docker virtiofs + Mac APFS の
ファイル名正規化問題)。

### 修正

`apps/api-admin/src/jobs/comicRootOps.ts`:

- `copyRecursive(JS)` を撤去し、**シェルの `cp -a <src>/. <dst>`** に置換
  - `cp -a` はパーミッション・タイムスタンプ・所有者を保持しつつ再帰コピー
  - 末尾 `/.` で「src の中身」を dst にコピー (src 自体を入れ子にしない)
- **コピー完全性検証**: `countFilesRecursive` で src/dst のファイル数を比較し、
  足りなければ src を削除せずに `ok=false` で停止
- `countFilesRecursive` は NFC/NFD のいずれでも到達できるよう `p.normalize`
  3バリアントを試行
- 移動成功時のメッセージにファイル数を含めて可視化
  例: `移動(cp -a): /comics/... → /regist/... (47/47 files)`

### 動作確認の指針

- 移動後の戻り値メッセージで `(47/47 files)` のように同数になっていれば完全
- 異なる数字なら警告 (`src は削除せず残しています`) が出る

## ER解凍: unrar-free のロケール問題と 1-bracket 形式への対応

### 不具合

実データで動かしたところ、以下2件で失敗:

1. **`unrar-free`** は非ASCIIファイル名で
   `Pathname cannot be converted from UTF-16BE to current locale` を吐いて解凍失敗
2. パーサが旧PHPの 2-bracket 形式 `[JpName] JpTitle - [EnName] EnTitle` しか受け付けず、
   実データの `[Author] RomajiTitle - EnglishTitle Digital` (1-bracket 形式) で
   `パース失敗` になる

### 修正

#### `apps/api-admin/src/jobs/extractErComic.ts` Part1 (解凍コマンド)

- **`7z x` を最優先**に変更 (zip/7z は当然、p7zip-full は RAR3 まで対応 / UTF-8 OK)
- `.rar` / `.cbr` で 7z が失敗した場合のみ `unrar-free` にフォールバック
- これにより英字+記号タイトルの rar も成功するようになる

#### Part2 (フォルダ名パース)

`parseERFolderName` を 3 つの形式に対応:

| 形式 | 入力例 | 出力 |
| --- | --- | --- |
| A) 2-bracket (旧PHP) | `[ぶるマほげろー] プリンプリンシプル - [Buruma Hogero] Pudding Principle` | enName/jpName が分離 |
| B) 1-bracket + dash | `[Ameyama Denshin] Anneliese no Takarabako - Anneliese's Treasure Chest Digital` | author を en/jp 共用、'-' でroma/英分離 |
| C) 1-bracket | `[Author] Title Digital` | author/title を en/jp 共用 |

事前に `[DL版]` `[Digital]` `Digital` 単語 `(完)` を除去、連続空白を圧縮。

### 動作確認 (B形式)

```
[Ameyama Denshin] Anneliese no Takarabako - Annelieses Treasure Chest Digital
↓
authorKey: "AmeyamaDenshin;Ameyama Denshin"
initial: "A"
newName: "[Ameyama Denshin] Anneliese no Takarabako - [AmeyamaDenshin] Annelieses Treasure Chest"
出力: /erc/A/AmeyamaDenshin;Ameyama Denshin/[Ameyama Denshin] Anneliese no Takarabako - [AmeyamaDenshin] Annelieses Treasure Chest/
```

### 通常の rar/zip 解凍にも同じ修正を適用

`extractAllArchives.ts` (rar/zip 解凍ボタン) の解凍コマンド分岐も同じ問題があったため、
ER と同じく:

- 1段目: `7z x` を最優先
- 2段目: `.rar/.cbr` で 7z が失敗した時のみ `unrar-free` にフォールバック

に変更した。これで「rar/zip 解凍」ボタンも非ASCII rar が解凍できるようになる。

## ER パーサ: 実データ形式に正しく対応

### 不具合

実フォルダ名 `[EnAuthor] EnTitle [Digital] [JpAuthor] JpTitle [DL版]` を
旧仕様の `[Jp] JpT - [En] EnT` (dash 区切り) と誤認していた。
結果として、

- 著者キーが `EnAuthor;EnAuthor` (両方英字) で生成
- 日本語部分が enTitle に混入
- initial がたまたま正しく出ても、フォルダ名構造が崩壊

### 修正

`parseERFolderName` を以下のルールに置き換え:

1. `[DL版]` `[Digital]` `(完)` を除去、空白圧縮
2. 正規表現 `^\[([^\]]+)\]\s*(.+?)\s*\[([^\]]+)\]\s*(.+)$` で
   2 ブラケット形式を非貪欲マッチ
3. `hasJapanese()` (ひらがな・カタカナ・CJK・半角カナ) で
   どちらが EN / JP かを判定 (順序の入れ替わりを許容)
4. enName は空白を詰めて `[EnNoSpace]` に
5. 出力フォルダ名: `[JpName] JpTitle - [EnNoSpace] EnTitle`
6. authorKey: `EnNoSpace;JpName`
7. initial: EnName 先頭1文字大文字

### 動作確認

| 入力 | 出力 |
|---|---|
| `[Ameyama Denshin] Anneliese no Takarabako - Anneliese's Treasure Chest [Digital] [雨山電信] あんねりーぜのたからばこ [DL版]` | `A/AmeyamaDenshin;雨山電信/[雨山電信] あんねりーぜのたからばこ - [AmeyamaDenshin] Anneliese no Takarabako - Anneliese's Treasure Chest` |
| `[Eco Heeky] Ero Biiky - You're my Favorite [Digital] [えこひいき] えろびいき [DL版]` | `E/EcoHeeky;えこひいき/[えこひいき] えろびいき - [EcoHeeky] Ero Biiky - You're my Favorite` |
| `[ぶるマほげろー] プリンプリンシプル - [Buruma Hogero] Pudding Principle` (legacy) | `B/BurumaHogero;ぶるマほげろー/[ぶるマほげろー] プリンプリンシプル - [BurumaHogero] Pudding Principle` |
