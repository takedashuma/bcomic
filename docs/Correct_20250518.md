## 1. 新着順 SQL 仕様の確定

ユーザー指定の参照SQLに合わせて comicFolders resolver を更新。

```sql
SELECT bok_vch6, bok_vch7,
       MAX(bok_txt1), MAX(bok_mid) AS max_bok_mid,
       MAX(bok_vch8) AS max_bok_vch8, COUNT(bok_vch8) AS cnt_bok_vch8
FROM tb_bok
WHERE bok_dlday IS NULL
  AND bok_vch9 = 'comic'
GROUP BY bok_vch6, bok_vch7
ORDER BY max_bok_mid DESC;
```

- GROUP BY を **日本語キー (authorJa=vch6, titleJa=vch7)** に変更
- WHERE に `vch9 = 'comic'` 追加
- ORDER BY を **MAX(id) DESC**（bok_mid の最大値降順）に変更
- お気入トグルもこの基準に合わせて日本語キーで動作するよう調整

## 5. Swagger UI からの CORS 対策

`openapi.yaml` の `servers:` 先頭に **`url: /`（相対）** を追加。Swagger UI を開いたのと同じオリジンへリクエストするので CORS の影響を受けない。

```yaml
servers:
  - url: /
    description: 現在開いているサーバ
  - url: http://192.168.40.99:4000
  - url: http://192.168.40.99:8080
  - url: https://dirs.synology.me
  - url: http://localhost:4000
```

## 7. 相対 URL 戦略（LAN/外部DDNS 両対応）

問題: ビルド時に焼き込まれた `VITE_API_URL` が固定IPだと、外部DDNS経由でアクセスした際に内部IPに繋ぎに行ってタイムアウト。

修正:
- `.env`: `VITE_API_URL=""`（空文字 = 相対URL）
- `docker-compose.yml`: `${VITE_API_URL-}` （`:-` を `-` に変更し、空文字を default で潰さない）
- `apps/web/src/lib/apollo.ts`: 既存ロジック（空ならアドレスバーの origin に投げる）を明文化

結果: LAN内アクセス（`192.168.40.99:8080`）でも DDNS（`dirs.synology.me`）でも同じビルドで動作。

## 12. COMIC_ROOT マウント問題対策

Mac Docker Desktop が `/Volumes/public` にアクセスできない問題に対し、3つの選択肢を docker-compose.yml と .env.example に整理:

- 対処A: Docker Desktop の File sharing に `/Volumes` 追加
- 対処B: NAS の SMB を Docker から直接マウント（`cifs` driver、`comic_data` named volume を用意）
- 対処C: `/Users` 配下に symlink

## 13. お気入トグルの即時反映（パフォーマンス）

旧: `refetchQueries: [COMIC_FOLDERS, FAVORITES]` でクリックごとに2クエリが再走行 → 体感遅い。

新: `optimisticResponse` + `cache.modify` で当該 ComicFolder の `isFavorite` だけキャッシュ書き換え。`cache.evict` で他クエリは次回参照時に再取得させる。

ファイル: `apps/web/src/components/FolderGrid.tsx`, `apps/web/src/routes/folder.tsx`

## 14. お気入トグル連打耐性

問題: 連打すると古い props で `next = !it.isFavorite` を計算 → server に並行リクエスト → race condition で重複行作成。

修正:
- クライアント: `pending` state で in-flight 中のクリックを破棄、ボタンに `disabled` + `opacity-50 cursor-wait`
- サーバ: `toggleFavorite` を `prisma.$transaction` でラップ + `updateMany` で全アクティブ行を一括 soft-delete

ファイル: `FolderGrid.tsx`, `folder.tsx`, `apps/api/src/schema/resolvers.ts`

## 15. 検索ボックス IME 対応

問題: 「おじさん」→「おjいsあn」のように IME 変換が途中で確定される。原因はキー入力ごとに navigate して再レンダー、Input value が更新されて composition が切れる。

修正:
- ローカル state `localQ` で値を持つ
- `onCompositionStart`/`onCompositionEnd` で composition フラグを管理
- composition 中は URL 更新スキップ、停止 300ms 後に navigate（debounce）

## 16. 検索ボックス Enter 即時実行

debounce 300ms を待たずに Enter で即検索。IME 変換確定の Enter は除外:

```ts
if (e.nativeEvent.isComposing || (e as any).keyCode === 229) return;
```

## 17. Favorites クエリ 504 対策

問題: 旧実装は `listFolders(onlyFavorites: true)` で内部に `WHERE vch9='comic' AND (favKeys のOR句が数百個)` の巨大 groupBy を実行 → tb_bok を全件スキャン → 504 タイムアウト。

修正: `favorites` を専用クエリに分離。tb_bkm から起点にして tb_bok を 1作品ずつピンポイントで引く実装に変更。インデックス（bok_vch6/vch7）が効くようになり数msに短縮。

ファイル: `apps/api/src/schema/resolvers.ts`

## 18. nginx proxy_read_timeout 180s

念のため `apps/web/nginx.conf` のグローバル proxy_read_timeout / send_timeout を 180s に延長。重いクエリでも 504 になりにくくする保険。

実装:
- ルート変更を検知して、タブ直下の URL なら search を **in-memory + sessionStorage** に保存
- ボトムナビボタンタップ時に保存先から navigate

ファイル: `apps/web/src/components/AppLayout.tsx`

## 20. 数値ページネーター + Top ボタン

`apps/web/src/components/Paginator.tsx` を改修。表示:

```
[Top] [前へ] [n-5]...[n-1] [n / total] [n+1]...[n+5] [次へ]
```

- Top: 常に1ページ目（検索条件は維持）
- 現在ページは青ハイライト + `n / total` 表示
- 端のページでは未存在番号は自動省略

## 21. ナビ復帰デグレ修正

問題1: 復帰時 URL が `?page="3"`（文字列クォート付き）になり一覧空。
原因: 旧 `serializeSearch` で `String(v)` していたため。

修正:
- `cleanSearch()` で **値の型(number/string)を保持**
- sessionStorage には JSON 経由（型は丸ごと維持）
- 空文字の `q` は除外

問題2: ボトムナビのアクティブ表示が更新されない。
原因: `useRouter()` は購読しないので、`router.state.location.pathname` は再レンダーされない。

修正:
- `useLocation()` でリアクティブに pathname を取得
- アクティブ時は `text-primary` + `font-medium` + アイコン線太め + `aria-current="page"`

問題3: 空文字の `q=` が URL に残る。
修正: `home.tsx`, `favorites.tsx` に `buildSearch()` ヘルパを導入し、`q === ""` のときは URL に含めない。
